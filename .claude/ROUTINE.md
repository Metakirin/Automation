## Filter criteria (applied in Step 1 of the routine prompt)

An issue is a candidate iff all of:

- It is an `Issue` (not a draft/note item on the board)
- `issue.state == "OPEN"`
- `"bug" ∈ issue.labels`
- Project field `Priority == "P1"`
- `"claude/triaged" ∉ issue.labels`
- Repository matches `Metakirin/Automation`
- No open PR whose body contains `Fixes #<issue-number>`
- No remote branch matching `claude/fix-<issue-number>-*`

## Verification command (inside the routine, before push)

```bash
npm ci && npm run test && npm run build
```

All three must exit 0 before the routine pushes or opens a PR. If any step
fails after one honest attempt to fix the fix, the routine abandons the
issue: deletes the local branch, does not push, logs the failure, moves on.

## Per-issue deliverables

1. Branch `claude/fix-<issue-number>-<slug>`
2. Commit `fix: <short title> (#<issue-number>)`
3. Draft PR with body:
   ```
   Fixes #<issue-number>

   <2-4 sentence root-cause + fix summary>

   ---
   Auto-generated by Claude routine.
   Session: https://claude.ai/code/${CLAUDE_CODE_REMOTE_SESSION_ID}
   ```
4. Comment on the issue linking the PR
5. Label the issue `claude/triaged`
6. (Optional — skip if field lookup fails) move Project item to "In review"
